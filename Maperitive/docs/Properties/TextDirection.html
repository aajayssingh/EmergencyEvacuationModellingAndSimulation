Maperitive 2.3.33 documentation

text-direction

Specifies the text direction for the text.

Possible values are:

  • auto : the text direction will be determined automatically (actually this currently means that the text direction will always be left-to-right (LTR)).
  • rtl : the text direction will be right-to-left.

Details

This property is a temporary workaround enabling support for right-to-left (RTL) scripts for linear texts (texts along the line). Maperitive needs to split linear texts into individual glyphs to be able to draw the text along a line so it cannot rely on the underlying graphics engine to determine the text direction.

The property does not have any effect on point and area texts since these are directed automatically by the underlying graphics engine.

Limitations

This workaround does not allow mixing RTL and LTR scripts in the same text value. Such texts will be rendered either fully LTR or fully RTL and thus incorrectly.

Full RTL support is planned in the near future.
